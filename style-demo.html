Recent Bug Fixes (December 2025)

Visual demonstrations of critical dropdown and chart rendering fixes implemented in feature/modern-ui-enhancements branch.

Fix #1: Dropdown Z-Index (99999)

Status: RESOLVED - Chart dropdowns now appear above all content without clipping

Problem: Dropdown menus inside chart boxes were clipping behind table cells, family navigator sections, and chart containers.

Root Cause:

  • Table cells with overflow: hidden created clipping contexts
  • Transform effects on hover created new stacking contexts
  • Original z-index (1050-1060) was insufficient

Solution: modern-enhancements.css lines 1527-1560

.wt-chart-box-dropdown,
.wt-chart-box-zoom-dropdown,
.wt-chart-box-links-dropdown {
    z-index: 99999 !important;
    position: absolute !important;
}

.wt-chart-box {
    overflow: visible !important;
}

.wt-facts-table td,
.wt-facts-table th,
.wt-facts-table tr {
    overflow: visible !important;
}
Before Fix
  • Dropdown clipped at table cell boundary
  • z-index 1060 insufficient
  • Content hidden behind parent containers
  • User unable to click dropdown items
After Fix
  • Dropdown fully visible with z-index 99999
  • Appears above all page content
  • All dropdown items clickable
  • Works in tables, charts, family navigator

Fix #2: Pedigree Connection Lines - All 4 Layouts

Status: RESOLVED - All pedigree orientations (RIGHT, LEFT, UP, DOWN) now display connection lines

Problem: Only RIGHT layout (default horizontal) showed connection lines between ancestor boxes. LEFT, UP, and DOWN layouts had no visual connectors.

Solution: modern-enhancements.css lines 1189-1250

RIGHT Layout (Horizontal ‚Üí)
  • Vertical lines on right edge of boxes
  • Rounded corners: 1rem 0 0 1rem
  • Lines extend 25% width
  • Height: 50% (centered on box)
LEFT Layout (Horizontal ‚Üê)
  • Vertical lines on left edge of boxes
  • Rounded corners: 0 1rem 1rem 0
  • Lines extend 25% width
  • Height: 50% (centered on box)
DOWN Layout (Vertical ‚Üì)
  • Horizontal lines at top of boxes
  • Rounded corners: 1rem 1rem 0 0
  • Lines extend 50% width (centered)
  • Height: 25%
UP Layout (Vertical ‚Üë)
  • Horizontal lines at bottom of boxes
  • Rounded corners: 0 0 1rem 1rem
  • Lines extend 50% width (centered)
  • Height: 25%

Fix #3: Vertical Chart Lines (Ancestors/Descendants)

Status: RESOLVED - Vertical connection lines added for Ancestors and Descendants charts

Solution: modern-enhancements.css lines 1310-1354

Vertical Chart Lines Features
  • 2rem indentation between generations
  • Border-left (1px solid) vertical line connecting parent to children
  • 1.5rem horizontal connector from vertical line to each child box
  • Rounded corner on last child: border-radius: 0 0 0 .25rem
  • Number styling: Sosa numbers (ancestors) and d'Aboville numbers (descendants) with dotted border
  • Expand/collapse controls functional for large family trees

Fix #4: Transform Effects Disabled on Dropdown Containers

Strategic Compromise: Transform hover effects disabled on elements with dropdown children to prevent clipping

Technical Background: CSS transforms create new stacking contexts, which clip child elements regardless of z-index.

Transform DISABLED
  • .wt-chart-box
  • .wt-facts-table tr
  • .wt-family-navigator-family

These elements have dropdown children, so lift effect removed but orange border + shadow expansion retained.

Transform ENABLED
  • .card
  • .wt-block
  • .img-thumbnail
  • .btn-primary

These elements have NO dropdown children, so full lift effect (translateY -1.5px) is safe.

Alternative Effects
  • Animated underline on links
  • Background color tint
  • Shadow expansion
  • Border color change

When transform is unsafe, use these effects for consistent hover feedback.
